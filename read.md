### 部署步骤（GitHub Pages）

```bash
cd "D:\Coworkcc\20260602web&app\trpg-static"
git add -A
git commit -m "人格页改版：世界任务+人格心声（手写信焚烧）+voices.md 心声库"
git push
```

- push 后 GitHub Pages 自动构建，**1-2 分钟**生效，无需其他操作。
- 线上地址：https://chain0321.github.io/trpg-shirengongzu/
- **首次部署后必须验证一项**：浏览器打开
  `https://chain0321.github.io/trpg-shirengongzu/voices.md`
  能看到心声原文（而不是 404），说明 `.nojekyll` 生效。若 404，确认 `.nojekyll` 已被提交（它是隐藏文件，容易漏）。
- 之后**日常改心声的流程**：编辑 `voices.md` → `git add -A && git commit -m "更新心声" && git push` → 等 1-2 分钟 → 强刷页面（Ctrl+F5）。页面 fetch 自带时间戳防缓存，正常刷新即可看到新内容。
- GitHub 被墙时的备用方案：base64 编码后走 `gh api` 上传（见 D:\Coworkcc\CLAUDE.md）。

## 2026-06-12 外来者 DLC 融入人格测试（v2 方案，推翻 v1）

**v1（已废弃）**：在原 24 题里硬插 8 个外来者选项，外来者不进雷达图。问题：所有题目都是团体视角，外来者选项像"在团体会议上说怪话的人"；雷达图上看不见外来者。

**v2（当前版本）**，与 PM 讨论后确定：

- **6 道「归属节点」题**（24 → 30 题，分布 2+2+1+1）：题目视角中立，不预设玩家在团体里，问的是"身位"——围绕组队/独行、中心/边缘、出走/归来、门内/门外四组母题。每题 5 个选项：3 个常规人格 + 1 个外来者A（被排斥的渴望者，+3分）+ 1 个外来者B（清醒的自我放逐，+3分）。六道题分别是：灾难那一刻你在哪里、走进别人的营地、棚子搭在哪、篝火夜坐在哪、出走三天后望着炊烟、新世界城门前。
- **雷达图改 12 轴**：外来者A/B 两轴放在圆环底部，虚线轴+红色标签，相邻环线也改虚线——视觉上是圆环开了一道"门缝"。
- **主导/影子全开放**：外来者可成为主导人格（有完整主导/影子文案）；团队定位新增「门外型」。影子人格规则：外来者必须得过分（至少选过一次外来者选项）才能成为影子，避免团体型玩家的影子永远是外来者。
- **DLC 卡片**：选中 ≥2 次外来者选项时出现，判定 A 型 / B 型 / 双重回响。
- 外来者A/B 角色现在映射到正式人格，通用「角色匹配分析」直接生效。
- 顺手修复：报告页「空间范围」描述把 HTML 标签当文字显示的旧 bug（改用 v-html）。

# 人格力场图 · 设计方案

> 状态：待确认 · 建议作为报告页的升级版可视化，替代或补充现有雷达图

---

## 一、为什么需要力场图

现有的雷达图和条形图告诉玩家「每个人格有多强」，但不回答一个更有趣的问题：

> 你内心的 10 个声音，谁跟谁是盟友，谁跟谁在对抗，谁被推到了角落？

力场图的目标是把测试结果从「体检报告」变成「心灵画像」——让玩家看到自己内心的动态关系，而不只是一组数字。

---

## 二、核心概念

### 2.1 一句话定义

一个圆形空间（「内心剧场」），10 个人格小人分布其中，通过位置、大小、亮度和连线来表达人格之间的活跃度与关系。

### 2.2 与雷达图的区别

| 维度 | 雷达图 | 力场图 |
|------|--------|--------|
| 回答的问题 | 每个维度多高 | 维度之间的关系 |
| 信息层次 | 单层（强度） | 双层（强度 + 关系） |
| 感受 | 像体检报告 | 像一幅心灵画像 |
| 叙事感 | 弱 | 强——能看到"法官和小丑在你心里对峙" |
| 技术实现 | Canvas 画多边形 | Canvas/SVG 力导向布局，稍复杂 |

## 三、人格关系矩阵

### 3.1 天然亲和（盟友）

来源：团队角色类型的组合逻辑。

| 关系对 | 类型 | 共通点 |
|--------|------|--------|
| 法官 ↔ 教授 | 锚点型 | 秩序 + 理性，用规则和逻辑稳定世界 |
| 暴君 ↔ 少年 | 引擎型 | 行动 + 信念，让事情真正发生 |
| 灵婆 ↔ 摆渡人 | 雷达型 | 感知 + 共情，最先感知到问题 |
| 小丑 ↔ X | 火花型 | 混沌 + 随机，打破僵局的变量 |
| 叛君 ↔ 史官 | 镜子型 | 揭示 + 记录，暴露被忽略的真相 |

### 3.2 天然对抗（张力）

来源：价值观的根本冲突。

| 关系对 | 冲突本质 |
|--------|----------|
| 法官 ↔ 小丑 | 规则 vs 打破规则 |
| 教授 ↔ 灵婆 | 逻辑分析 vs 直觉感应 |
| 暴君 ↔ 摆渡人 | 控制执行 vs 放手共情 |
| 少年 ↔ 史官 | 改变未来 vs 记录过去 |
| 叛君 ↔ 法官 | 揭面具 vs 维护秩序 |

## 四、视觉设计

### 4.1 空间布局

圆形空间，10 个人格按固定角度均匀分布（类似钟面）：

```
              · 教授 ·
        法官 ·         · 史官
                 ★ 
      暴君 ·    (你)    · 灵婆
                  
        少年 ·         · 摆渡人
              · 叛君 ·
          X ·           · 小丑
```

固定位置的好处：亲和对在相邻位置，对抗对在对角位置，视觉上自然形成「盟友群落」和「张力轴线」。

### 4.2 人格节点的视觉编码

| 属性 | 活跃人格（高分） | 沉睡人格（低分） | 影子人格 |
|------|----------------|----------------|---------|
| 大小 | 放大（1.5x-2x） | 缩小（0.6x） | 正常大小 |
| 亮度 | 正常/发光 | 变灰（opacity 0.3） | 半透明虚影 |
| 位置 | 靠近中心 | 推向边缘 | 边缘，带特殊标记 |
| 标注 | 名称 + 分数 | 仅名称 | 名称 + "影子" |

### 4.3 连线的视觉编码

| 关系 | 两者都活跃 | 一强一弱 | 两者都弱 |
|------|----------|---------|---------|
| 亲和 | 亮实线 + 微光 | 淡实线 | 不显示 |
| 对抗 | 红色虚线 + 张力标记 | 淡虚线 | 不显示 |

### 4.4 参考风格

- 整体调性：与十人小组一致（红 #C8362C + 黑 #1a1a1a + 米白 #FAFAF7）
- 可参考的视觉类型：
  - 力导向图（D3.js force layout）
  - 星座图（节点 + 连线）
  - 「欲望彩虹」那种人物关系的空间分布

## 五、交互设想

### 5.1 静态版（v1，建议先做）

- 固定布局，无动画
- 根据测试分数调整节点大小和透明度
- 显示亲和线和对抗线
- hover 节点显示该人格的一句话描述

### 5.2 动态版（v2，后续升级）

- 加载时有「聚合」动画——节点从散乱到各就各位
- 点击节点展开该人格的详细卡片
- 对抗线上可以显示一句叙事文案，例如：
  - 法官↔小丑：「你心里最安静的角落，住着一个想掀桌子的声音」
  - 教授↔灵婆：「逻辑告诉你一个答案，直觉告诉你另一个」

## 六、技术方案

### 6.1 推荐实现

- **Canvas 或 SVG**，纯前端，无依赖
- 10 个节点的位置可以预计算（固定角度），不需要真正的力导向物理模拟
- 节点大小 = `baseSize * (score / maxScore)`
- 节点到中心的距离 = `maxRadius * (1 - score / maxScore)` （分数越高越靠近中心）

### 6.2 数据输入

```javascript
// 力场图需要的数据
const forceFieldData = {
  // 来自测试结果
  scores: { judge: 9, prof: 14, tyrant: 2, ... },
  shadowKey: 'tyrant',
  
  // 固定关系矩阵
  alliances: [['judge','prof'], ['tyrant','youth'], ['shaman','ferry'], ['joker','x'], ['rebel','scribe']],
  tensions: [['judge','joker'], ['prof','shaman'], ['tyrant','ferry'], ['youth','scribe'], ['rebel','judge']],
};
```

### 6.3 工作量估算

| 阶段 | 内容 | 预估 |
|------|------|------|
| v1 静态版 | 固定布局 + 节点大小/透明度 + 连线 | 3-4 小时 |
| v2 动画版 | 加载动画 + hover 交互 + 叙事文案 | 额外 3-4 小时 |

## 八、一个完整示例

> 玩家小明：选了「法官」，测出主导人格「教授」，影子人格「暴君」

他的力场图应该呈现：

- **教授**：最大、最亮，靠近中心
- **法官**：第二大，靠近中心，与教授之间有一条亮线（亲和·锚点型）
- **暴君**：最小、半透明虚影，在边缘，标注「影子」
- **教授↔灵婆**：有一条淡红虚线（对抗），灵婆在中等位置
- 其余人格按分数分布，低分的灰掉

角色匹配提示浮在力场图下方：
> 你选择了法官，但你的内心剧场中心站着教授。法官是你想成为的人，教授是你本能的反应方式。
